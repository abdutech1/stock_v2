Open the first task in To Do ("Auth & Multi-tenancy Middleware").
Click Convert to issue (this turns the draft into a real GitHub Issue linked to your repo — highly recommended).
Add more details if needed (acceptance criteria, notes).
Move it to In Progress.
Start working on it!

Add useful fields to make tracking better (Highly recommended)
In your project, click … → Manage fields → New field
Add these custom fields:
Priority (Single select: High, Medium, Low)
Estimated effort (Number: in hours or story points, e.g., 4, 8, 16)
Due date (Date field) — optional for now

Then drag these fields into your board view so you can see them on each card.

Set up simple Automation (saves you time)
Go to your project → … → Workflows or Automations
Turn on these if available:
When an issue is closed → automatically move to Done
When status changes → move between columns (GitHub can do basic ones automatically)

Recommended Weekly Workflow (Simple & Sustainable)

Every Monday: Review Backlog → Move next 4–6 tasks to To Do
Daily:
Move one task to In Progress when you start coding
Update the card (add comments, link PRs)

When you finish coding + testing: Move to Review/QA
When everything is verified: Move to Done
End of week: Look at how many tasks moved to Done. Celebrate!
